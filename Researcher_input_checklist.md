# Researcher Input Checklist
*For user researchers adding data to the insights repository*

---

## 1. Before Collecting Data
- [ ] Confirm participants understand what will be collected and how it will be used.
- [ ] Make sure consent or lawful basis is recorded (usually “public task” for councils).
- [ ] Plan how quotes will be anonymised before being shared or stored.

---

## 2. Before Uploading to the Repository
- [ ] Remove names, organisation names, or anything that can identify participants.
- [ ] Reword quotes slightly if they contain identifying details but keep the meaning.
- [ ] Label your upload with the study title, date, and participant group.
- [ ] Choose a confidence level (Low / Medium / High).

---

## 3. During Upload
- [ ] Use the correct template (insight, quote, or study summary).
- [ ] Add relevant tags: user group, need, theme, and journey stage.
- [ ] Mark if anonymisation is “full” or “partial.”
- [ ] Link your file to its original study if possible.

---

## 4. After Upload
- [ ] Check your entry appears in the index (searchable).
- [ ] Ask a peer or lead researcher to review for sensitive data.
- [ ] Flag any insights that need further anonymisation.

---

## 5. Good Data Hygiene
- [ ] Don’t upload raw transcripts, video, or identifiable notes.
- [ ] Delete local copies of sensitive data once it’s stored securely.
- [ ] Review old insights every 12 months
